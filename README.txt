---------------------------------------------------
Установка
---------------------------------------------------
1. Установить Python с сайта https://www.python.org/
2. Перейти в директорию с проектом ML, открыть cmd (Windows) или терминал (MacOS, Linux) в данной директории и ввести:

pip install -r requirements.txt

---------------------------------------------------
Требования к данным
---------------------------------------------------
1. Данные для обучения и теста должны лежать в директории ML/data/
2. Данные для обучения должны находиться в файле data_train.csv
3. В данных для обучения обязательно должен присутствовать столбец 'timeframe' и называться он должен именно 'timeframe'.
4. Также в данных для обучения должен присутствовать столбец 'result' и называться он должен именно 'result',
не 'Result', не 'RESULT' и т.д.
5. Данные для теста должны храниться в файле data_test.csv.
6. В данных для теста должен присутствоать столбец 'timeframe', который так и называется.
7. В данных для теста НЕ должен присутствовать столбец 'result'. Т.е. это именно те данные, на которых мы в боевом
режиме испытываем работу моделей.
8. В колонке 'result' два допустимых значения: 1 и -1

---------------------------------------------------
Обучение
---------------------------------------------------
Для обучения перейти в консоли в директорию ML/ и набрать:

python train.py

---------------------------------------------------
Тест
---------------------------------------------------
Для теста перейти в консоли в директорию ML/ и набрать:

python test.py

---------------------------------------------------
Эксперименты
---------------------------------------------------
При экспериментах с признаками, их количеством и т.д. ОБЯЗАТЕЛЬНО:

1. Тестировать модель именно на таких же данных, что и обучали, то есть с таким же составом и порядком столбцов
2. Если возникнет необходимость экспериментов, обязательно нужно

а) Перестроить данные для трейна и теста, чтобы состав столбцов и их порядок были одинаковыми, за исключением последнего
столбца 'result', который присутствует в трейне и не присувует в тесте
б) Обучить модели на новых данных
в) И только после этого запускать тест

---------------------------------------------------
Модели
---------------------------------------------------
Сохраненные модели располагаются в корне проекта ML/ в следующих файлах:

strict_model.sav
middle_model.pickle.dat
risk_model.sav

---------------------------------------------------
Описание моделей
---------------------------------------------------
Модели представляют собой:

1. Строгая - логистическую регрессию, которая предсказывает неудачу. Есть возможность давать вероятность успеха или неудачи
2. Средняя - градиентный бустинг, который предсказывает неудачу
3. Рисковая - логистическую регрессию с измененным балансом классов, которая предсказывает успех. Есть возможность
давать вероятность успеха или неудачи
